# Astrbot Deltaforce

三角洲行动 AstrBot 综合查询、战报、工具与推送插件。

## 简介

本插件面向 AstrBot 用户，提供三角洲行动相关的账号登录、个人数据查询、战报分析、价格工具、特勤处信息、开黑房间、改枪方案、TTS/语音/音乐以及群内推送等能力。

当前仓库中的命令入口已经覆盖账号管理、数据查询、社区功能和日常推送等多个场景，适合在群聊或私聊中快速完成常用操作。

## 安装

1. 将本插件放入 AstrBot 的 `plugins` 目录。
2. 安装 Python 依赖：

```bash
pip install -r requirements.txt
playwright install chromium
```

3. 如果运行环境是 Linux 或 Docker，按需补充系统依赖：

```bash
playwright install-deps
```

4. 在 AstrBot 管理面板中配置 `token` 与 `clientid`。
5. 重启 AstrBot。

> `apscheduler` 与 `aiosqlite` 由 AstrBot 宿主环境提供，无需单独安装。

## 配置项说明

配置定义位于 `_conf_schema.json`，建议至少完成以下核心配置：

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `token` | API Token，通常从 [df.shallow.ink](https://df.shallow.ink/) 的 API 管理页获取 | - |
| `clientid` | 后端用户 ID，通常从 [df.shallow.ink](https://df.shallow.ink/) 个人中心获取 | - |
| `api_mode` | API 请求模式，支持 `auto`、`default`、`eo`、`esa`，推荐使用 `auto` | `auto` |
| `api_timeout` | API 请求超时时间，单位秒 | `30` |
| `api_retry_count` | API 请求失败后的重试次数 | `3` |
| `push_daily_keyword_enabled` | 是否启用每日密码定时推送 | `false` |
| `push_daily_keyword_cron` | 每日密码推送的 Cron 表达式 | `0 8 * * *` |
| `push_daily_keyword_groups` | 每日密码推送群号列表，多个群用逗号分隔 | 空 |
| `push_daily_report_enabled` | 是否启用日报定时任务 | `false` |
| `push_daily_report_cron` | 日报推送的 Cron 表达式 | `0 10 * * *` |
| `push_weekly_report_enabled` | 是否启用周报定时任务 | `false` |
| `push_weekly_report_cron` | 周报推送的 Cron 表达式 | `0 10 * * 1` |
| `push_place_task_enabled` | 是否启用特勤处制造完成推送 | `true` |
| `broadcast_admin_users` | 可执行广播命令的管理员用户 ID，多个用逗号分隔 | 空 |
| `broadcast_default_targets` | 广播默认目标群号，多个用逗号分隔 | 空 |

### 关于 `api_mode`

- `auto`：自动切换可用接口，推荐日常使用
- `default`：固定使用默认接口
- `eo`：固定使用 `eo` 接口
- `esa`：固定使用 `esa` 接口

## 快速开始

推荐按下面的顺序完成首次使用：

1. 先登录一个可用账号，例如 `/三角洲QQ登录`、`/三角洲微信登录` 或 `/三角洲WeGame登录`
2. 执行 `/三角洲账号列表` 查看当前已绑定账号
3. 如果绑定了多个账号，使用 `/三角洲切换 1` 激活目标账号
4. 再执行 `/三角洲信息`、`/三角洲战绩`、`/三角洲日报` 等个人数据命令

## 推送、权限与前置条件

### 推送规则

- `每日密码推送` 只能在群聊中开启或关闭，且通常需要群管理员操作
- `日报推送` 和 `周报推送` 也是在群聊中订阅，但属于用户维度订阅，订阅前需要先绑定并激活账号
- `特勤处推送` 用于制造完成提醒，是否可用取决于推送模块与相关配置是否初始化
- `推送状态` 可查看当前调度器和任务启用情况

### 权限说明

- `更新日志` 与 `插件状态` 依赖 AstrBot 的管理员权限控制
- `广播` 与 `广播历史` 额外依赖配置项 `broadcast_admin_users`
- 帮助页文案中部分系统项显示为“仅主人”，实际以 AstrBot 权限系统和插件配置为准

### 使用前置

- 大多数个人查询命令依赖当前激活账号
- `违规历史` 需要安全中心登录态
- 图文渲染功能依赖 Playwright；未正确安装时会自动回退为纯文本回复
- `大红收藏`、个人数据、战报、帮助页、部分音乐列表等能力都可能走图片渲染
